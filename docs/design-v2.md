## 2. 競合状況と差別化

### 競合マップ

| 競合タイプ | 代表例 | 弱点（＝ユーザーの不満） |
|---|---|---|
| 企業メディア | uyet media, d-gear.biz | 「自社ツールを買わせるための記事」感が強い。初心者に威圧的 |
| 個人ブログ | ess-centre.org, coro-vtuber.com | AI生成感・テンプレ感、情報が古い、独自性なし |
| プラットフォーム公式 | coconala, Palmie | 自社サービスへの誘導が露骨すぎる |
| 英語圏ガイド | VTuber Sensei, Streamlabs Blog | 英語のみ、日本のプラットフォーム（BOOTH/nizima）は未対応 |

### ユーザーが選ぶ理由（ユーザー視点での差別化）

> ⚠️ 「ガイドサイト + 自社商品販売」は運営者視点の優位性であり、ユーザーにとってのメリットにはならない。
> ユーザーは「なぜここを読むべきか」を必要としている。

**ユーザーに刺さる差別化3点**

1. **現役VTuberが書いている**
   - メリノちゃんが実際に活動しながら得た知見をベース
   - 「モデル費用いくらかかった？」→ 実体験から答えられる（机上の比較記事と違う）

2. **海外情報を日本語で読める唯一のハブ**
   - 英語圏の優良ツール（VSeeFace, Live3D, itch.io素材等）は日本語情報がほぼゼロ
   - 海外VTuberの成功事例・費用相場・Twitch活用法なども日本語で解説
   - 競合はほぼ全員「国内向け」に閉じており、海外情報はほぼ空白

3. **特定サービスへの誘導圧力がない**
   - CyberLink, coconalaなどは自社製品への誘導が目的の記事ばかり
   - このサイトは「何を選んでも良い」フラットな比較が可能
   - （BOOTH/nizimaへのCTAは記事文脈に沿った場合のみ）

## 5. 海外情報活用戦略（新設）

### なぜ海外情報が強いか

MDXベースの記事システムにより、英語原文を調査→日本語で構造化→記事生成のフローが確立しており、海外情報の記事化コストはほぼゼロ。

一方、競合（uyet media, スコマガ等）は国内情報に終始しており、海外のVTuberシーンはほぼ無視されている。これはコンテンツ空白地帯。

### 海外情報ソース（信頼度順）

| ソース | 内容 | 活用方法 |
|---|---|---|
| VTuber Sensei (WordPress) | ビジネスモデル・費用・産業分析 | 日本語意訳+独自分析追加 |
| Virtual YouTuber Wiki | ソフトウェア・ツール総合リスト | ツール紹介記事のリサーチ元 |
| Streamlabs Blog | 初心者向けセットアップ手順 | 海外標準構成の紹介 |
| VTuber Sensei 費用分析 | 英語圏初期費用 $100〜$10,000 | 「海外との費用比較」記事に使用 |
| GitHub VTuber Software List | オープンソース全ツール | 「知られていないツール」記事 |
| r/VirtualYoutubers | コミュニティの生の声 | トレンド把握・悩みリサーチ |
| Stream Hatchet レポート | Twitch/YouTube シェア分析 | データ引用（出典明記） |

### 海外コンテンツの作り方（ルール）

- **ファクトチェックを入れる**: 調査元URLを記事内（非表示でもFrontmatterに）記録
- **翻訳記事はNG**: 「海外ではこう」という日本人目線の解説記事として書く
- **古くなりやすい情報に年号を入れる**: 「2026年版」「2026年時点」

### 将来の英語展開（中期）

Next.jsの`[locale]`ルーティングを使えば英語版ページの追加はほぼゼロコスト。
潜在市場：
- 「How to start VTubing」系 月50万検索（英語圏）
- インドネシア・中国でも独立VTuber市場が急成長中

## 9. 記事品質保証システム（QA）

### 基本方針

> **「嘘をつかない記事だけ公開する」**
> 検証できない情報・スクショが必要な手順・未使用機材のおすすめは掲載しない。
> メリノ工房の信頼性は記事の正確性に直結する。

---

### レビューワークフロー（全記事必須）

```
Step 0: 記事の分類宣言
  → カテゴリ / 情報ソース種別 / スクショ要否 を確認
  → スクショ必要 → 作らない
  → 未検証機材 → 作らない
         ↓
Step 1: リサーチフェーズ
  → 国内ソース調査 → 情報源URLをFrontmatterに記録
  → 海外ソース調査（必要な場合）→ 同上
  → 価格・手数料は「何月時点」かを確認
         ↓
Step 2: 草稿生成
  → MDX記事を生成
  → Frontmatterのsourcesフィールドに全ソースURL記入
         ↓
Step 3: ファクトチェック30項目（下記）
  → 全項目をチェックリスト形式で確認
  → NG項目は修正 or 記述を削除
         ↓
Step 4: 文体・ブランドレビュー
  → メリノちゃん人格ルール確認
  → CTA妥当性確認
         ↓
Step 5: 技術的チェック
  → MDXビルドエラーなし確認
  → 全外部リンク疎通確認
         ↓
Step 6: 公開承認
  → 社長の目視確認（新カテゴリ記事・海外情報記事・税務記事は必須）
  → 通常記事はメリノちゃんが最終判断可
```

---

### ファクトチェック30項目

記事公開前に全項目を確認する。NG箇所は修正 or 記述削除。

#### A. 情報ソース（6項目）

| # | チェック内容 |
|---|---|
| A-1 | 記事で述べた全ての数値・事実にソースURLがある |
| A-2 | ソースが90日以内の情報、または「○年○月時点」と明記している |
| A-3 | 海外ソースを使った場合、原文と日本語訳の間に意味の乖離がない |
| A-4 | 「〜と言われている」「〜らしい」等の曖昧表現を使っていない |
| A-5 | 1次ソース（公式サイト・公式発表）を少なくとも1つ以上含む |
| A-6 | Wikipedia・まとめサイト・個人ブログのみを根拠にしていない |

#### B. 価格・費用情報（5項目）

| # | チェック内容 |
|---|---|
| B-1 | 価格は「○年○月時点」または「調査時点」と明記している |
| B-2 | 価格範囲（〜円〜〜円）の根拠ソースが存在する |
| B-3 | メリノちゃん自身の制作費を「相場」として使っていない（自社制作は参考外） |
| B-4 | 手数料・税率は2026年現在の正式な数値である |
| B-5 | 「無料」と書いたサービスが現在も無料であることを確認した |

#### C. プラットフォーム・サービス情報（5項目）

| # | チェック内容 |
|---|---|
| C-1 | 各プラットフォームの公式URLが正しい（リダイレクト先も確認） |
| C-2 | サービスの仕様・機能説明が公式ページと一致している |
| C-3 | 廃止・統合されたサービスを現役として紹介していない |
| C-4 | プラットフォーム名に正式なURLリンクを付与している |
| C-5 | 海外ツール紹介時、日本からのアクセス可否・日本語対応状況を確認した |

#### D. 法務・税務情報（4項目）

| # | チェック内容 |
|---|---|
| D-1 | 税務情報は「2026年現在」であることを確認し、変更可能性の注意書きがある |
| D-2 | 「必ずこうしなければならない」と断言していない（「一般的に〜」等に留める） |
| D-3 | 税理士・弁護士への相談推奨の一文がある（税務・法務記事） |
| D-4 | 著作権・契約に関する記述は「実務上の一般論」に限定し、個別事案の判断は除外 |

#### E. MDX・技術品質（5項目）

| # | チェック内容 |
|---|---|
| E-1 | `npm run build` または開発サーバーでMDXエラーが出ていない |
| E-2 | ComparisonTable / StepGuide 等のコンポーネントが正しくレンダリングされている |
| E-3 | MDXコンポーネント内のboldは `<strong>` を使用（`**` はNG） |
| E-4 | frontmatterの `title` / `description` / `category` / `tags` が全て埋まっている |
| E-5 | frontmatterに `sources` フィールドが存在し、ソースURLが1件以上記録されている |

#### F. ブランド・文体（5項目）

| # | チェック内容 |
|---|---|
| F-1 | 「メリノ」呼び捨てを使っていない（必ず「メリノちゃん」） |
| F-2 | メリノちゃんが「AIです」「調べました」等のAI自称をしていない |
| F-3 | 肉体的体験（食べた、触った等）の直接描写をしていない |
| F-4 | CTACard / CTABanner が記事内容と無関係なモデルを紹介していない |
| F-5 | 競合他社・個人VTuberの批判・比較ランキングを含んでいない |

### Frontmatter 品質仕様（必須フィールド）

```yaml
---
title: "記事タイトル（〜2026年版〜 等の年号推奨）"
description: "160文字以内のSEO用メタディスクリプション"
category: "model | software | management | getting-started | equipment"
tags: ["タグ1", "タグ2"]
publishedAt: "YYYY-MM-DD"
updatedAt: "YYYY-MM-DD"
sources:
  - url: "https://..."
    title: "ソース名"
    accessedAt: "YYYY-MM-DD"
  - url: "https://..."
    title: "ソース名"
    accessedAt: "YYYY-MM-DD"
factcheck: "YYYY-MM-DD"  # チェック実施日（必須）
needsReview: false       # true の場合は社長確認待ち
---
```

`sources` と `factcheck` が空の記事は公開しない。

### よくある失敗パターン（過去事例より）

| パターン | 原因 | 対策 |
|---|---|---|
